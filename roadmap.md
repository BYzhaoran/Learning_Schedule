# 两年DRL工程师学习地图（按关键部分重构版）

定位：
- 风格：先补基础，再做闭环，再做融合，最后做成果收敛
- 强度：每天5小时
- 主线优先级：RL/DL与控制双主线并进，ROS2与仿真作为系统集成主线
- 约束：每月必须有可复现实验结果与可展示产物

## 1. 总目标（24个月）

24个月后达到：
- 至少独立实现并稳定训练3类DRL算法（DQN、PPO、SAC/TD3）
- 能独立完成PID/LQR并理解MPC，形成统一评估体系
- 能搭建感知-决策-控制闭环（ROS2+仿真+控制/RL可切换）
- 形成2个完整项目包（代码+报告+视频+复现实验）

## 2. 大纲映射（不改权重）

| 关键部分 | 占比 | 角色定位 | 在学习顺序中的位置 |

## 3. 内在联系与先后顺序

核心依赖链：
- 数学 + Python/C++ -> 控制理论/DRL实现 -> ROS2系统集成 -> Sim2Real -> VLA与成果收敛
- 视觉是中间层能力：为ROS2感知节点、RL状态表示、VLA输入提供统一基础
- 嵌入式在“能跑通系统后”发力，避免前期被硬件细节拖慢算法主线

并行策略：
- 主线A（算法）：RL/DL -> DRL训练与调参 -> 融合策略
- 主线B（系统）：控制 -> ROS2 -> 仿真与真机
- 辅线C（研究）：论文阅读全程跟进，驱动阶段选题和实验设计

## 4. 24个月路线（按依赖重排）

### 阶段1（M1-M3）基础重建期

目标：建立可编码、可实验、可复盘的最低工程能力。

- M1 数学速成 + Python基础
	- 重点：线代、概率、优化；Numpy/PyTorch基本训练循环
	- 门槛：手写线性回归，解释学习率与收敛行为
- M2 C++与工程习惯
	- 重点：STL、CMake、调试、Git协作
	- 门槛：完成小型数据结构/工具库，修复10个以上真实bug
- M3 训练工程模板
	- 重点：配置管理、日志、实验记录、结果复现
	- 门槛：训练仓库模板可复用（训练/评估/可视化）

对应大纲：数学、Python/C++、论文阅读（入门）

### 阶段2（M4-M6）控制与机器人基础期

目标：先掌握可解释控制，再进入学习型控制。

- M4 建模与PID
	- 门槛：同一对象完成PID整定，输出指标表
- M5 状态空间与LQR
	- 门槛：完成LQR仿真并解释Q/R变化影响
- M6 MPC入门与控制对比
	- 门槛：输出PID vs LQR报告，补充MPC最小可运行示例

对应大纲：控制理论+运动学动力学、数学、论文阅读

### 阶段3（M7-M9）视觉与DL工程期

目标：从“会调用模型”升级到“会训练、会诊断、会迭代”。

- M7 PyTorch训练模板完善
	- 门槛：训练/验证/保存/恢复/可视化全流程跑通
- M8 视觉任务闭环（如YOLO）
	- 门槛：完成训练、错例分析、再训练闭环
- M9 数据与训练自动化
	- 门槛：数据-训练-评估自动化脚本v1

对应大纲：Computer Vision、RL/DL基础、Python工程

### 阶段4（M10-M12）DRL核心能力期

目标：从经典控制扩展到学习控制，建立调参与消融习惯。

- M10 DQN
	- 门槛：CartPole稳定过阈值，日志完整
- M11 PPO
	- 门槛：至少1个任务稳定训练并完成消融
- M12 SAC/TD3 + MuJoCo任务封装
	- 门槛：可训练、可回放、可导出

对应大纲：RL/DL理论与实践、论文阅读、Sim2Real预热

### 阶段5（M13-M15）ROS2系统集成期

目标：形成系统闭环，不再停留在单算法演示。

- M13 ROS2基础工程
	- 门槛：多节点通信稳定（topic/service/action）
- M14 感知-控制闭环
	- 门槛：视觉输出驱动控制演示
- M15 ROS2 + 仿真协同
	- 门槛：RL与控制器统一接口可切换

对应大纲：ROS2、CV、控制、RL

### 阶段6（M16-M18）融合与Sim2Real准备期

目标：从“能跑”升级到“能比较、能融合、能落地”。

- M16 DRL与控制器标准化对比
	- 门槛：20组以上对比实验
- M17 融合策略实现
	- 门槛：DRL+控制保护层v1上线
- M18 鲁棒性与域随机化
	- 门槛：Sim2Real实验框架v1

对应大纲：Sim2Real、控制+RL融合、论文阅读

### 阶段7（M19-M21）工程化与嵌入式强化期

目标：提升可复现性、性能与硬件落地能力。

- M19 一键工具链
	- 门槛：训练/评估/导出一键化
- M20 测试体系
	- 门槛：单测+集成+回归链路v1
- M21 性能与部署优化
	- 门槛：吞吐较基线提升30%+

并行加入：嵌入式驱动、传感器接入、边缘部署

### 阶段8（M22-M24）VLA与成果收敛期

目标：形成前沿方向能力与职业化交付。

- M22 VLA/具身前沿复现
	- 门槛：复现1篇核心论文并给出落地结论
- M23 系统冻结
	- 门槛：v2.0全链路稳定演示
- M24 作品集封装
	- 门槛：2个完整项目包（代码+报告+视频+讲解）

对应大纲：VLA、论文阅读、系统化表达

## 5. 每天5小时执行模板

- 90分钟：单主题学习（当天只学1个核心概念）
- 120分钟：唯一代码任务（防止任务分散）
- 60分钟：至少1组对比/消融实验
- 30分钟：复盘（结论、问题、明天第一步）

当日红线：
- 1次可复现实验
- 1条明确结论
- 1次代码提交

## 6. 每周执行模板

- 周一：定义唯一目标，跑基线
- 周二-周三：编码与训练
- 周四：对比/消融
- 周五：复盘、修bug、写周报
- 周六：补短板（可选2-4h）
- 周日：休息或轻阅读

## 7. 防跑偏规则

- 本周无实验结果，下周学习时间减30%，转实现与调参
- 同一问题卡3小时，立刻降级为最小可运行版本
- 每周至少2次盲写核心逻辑
- 每月底只检查一件事：本月门槛是否达成
