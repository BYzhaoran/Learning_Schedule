# 两年DRL工程师学习地图（按关键部分重构版）

定位：
- 风格：先补基础，再做闭环，再做融合，最后做成果收敛
- 强度：每天5小时
- 主线优先级：RL/DL与控制双主线并进，ROS2与仿真作为系统集成主线
- 约束：每月必须有可复现实验结果与可展示产物

## 1. 总目标（24个月）

24个月后达到：
- 至少独立实现并稳定训练3类DRL算法（DQN、PPO、SAC/TD3）
- 能独立完成PID/LQR并理解MPC，形成统一评估体系
- 能搭建感知-决策-控制闭环（ROS2+仿真+控制/RL可切换）
- 形成2个完整项目包（代码+报告+视频+复现实验）

## 2. 大纲映射（不改权重）

| 关键部分 | 占比 | 角色定位 | 在学习顺序中的位置 |
| --- | --- | --- | --- |

## 3. 内在联系与先后顺序

核心依赖链：
- 数学 + Python/C++ -> 控制理论/DRL实现 -> ROS2系统集成 -> Sim2Real -> VLA与成果收敛
- 视觉是中间层能力：为ROS2感知节点、RL状态表示、VLA输入提供统一基础
- 嵌入式在“能跑通系统后”发力，避免前期被硬件细节拖慢算法主线

并行策略：
- 主线A（算法）：RL/DL -> DRL训练与调参 -> 融合策略
- 主线B（系统）：控制 -> ROS2 -> 仿真与真机
- 辅线C（研究）：论文阅读全程跟进，驱动阶段选题和实验设计

## 4. 24个月路线（按依赖重排）

### 阶段1（M1-M3）基础重建与控制基础期

目标：建立可编码、可实验、可复盘的最低工程能力 + 控制论直觉。

**时间压缩亮点：M1-M2 合并完成原 M1-M3 的核心，提前1个月进入控制领域**

- M1 数学与Python工程基础速成（6周）
	- W1-2：线性代数 + 概率 + Numpy
	- W3-4：C++ 与数据结构库
	- W5-6：工程工作流与可复现性
	- 门槛：手写线性回归、修复bug、工程模板可用

- M2 深度学习与训练工程速成（6周）
	- W1-2：PyTorch 完整训练循环 + 排错SOP
	- W3-4：分类任务 + 数据增强 + 错例分析
	- W5-6：工程规范与快速迭代框架
	- 门槛：单模型稳定训练、新任务30分钟接入

- M3 控制论基础入门
	- 重点：系统建模、PID控制、性能指标
	- 门槛：同一对象完成PID调参，输出指标对比表

对应大纲：数学、Python/C++、深度学习基础、控制论基础、论文阅读（入门）

### 阶段2（M4-M6）控制深化与数据驱动思维期

目标：掌握从PID→LQR→MPC的控制演进路线，为DRL奠定思想基础。

**合并亮点：M4-M5 合并为单一对象上的PID+LQR对比，M6 新增作为控制→DRL桥梁**

- M4 PID + LQR 控制系统完整设计（合并原M4-M5）
	- W1：建模与PID深入（反饱和、离散化）
	- W2：LQR实现与5组Q/R配置对比
	- 门槛：能清楚说明PID与LQR各自优缺点，独立设计LQR

- M5 MPC与约束优化控制
	- 重点：滚动优化、约束处理、QP求解
	- 门槛：PID vs LQR vs MPC 三方案对比，含约束场景分析

- M6 系统辨识与数据驱动学习
	- 重点：参数辨识（LS）vs 黑盒学习（神经网络）
	- 门槛：形成"参数模型 vs 黑盒学习"的权衡认识

对应大纲：控制理论+运动学动力学、系统辨识、神经网络基础、论文阅读

### 阶段3（M7-M9）深度学习工程加速期

目标：从"会调用模型"升级到"会设计、会训练、会诊断、会快速迭代"。

**加速亮点：M7-M8 压缩至5周（从8周），M9 改为基础架构设计而非工程自动化**

- M7 PyTorch训练体系加速版（3周核心）
	- 重点：优化器对比、学习率调整、混合精度、排错能力
	- 门槛：30分钟快速接入新数据、15分钟定位NaN/显存溢出

- M8 计算机视觉任务加速版（2周迭代）
	- 重点：分类基线→错例分析→针对性改进的快速循环
	- 门槛：2周内形成数据-训练-分析-改进完整闭环

- M9 CNN架构与深度网络设计基础（4周）
	- 重点：VGG→ResNet演进、MobileNet轻量化、SE-Block注意力
	- 门槛：理解架构演进原则，能评估模型的精度-FLOPs权衡

对应大纲：Computer Vision、深度网络设计、RL/DL基础、Python工程

### 阶段4（M10-M12）DRL核心能力期

目标：从经典控制扩展到学习控制，建立调参与消融习惯。

- M10 DQN
	- 门槛：CartPole稳定过阈值，日志完整
- M11 PPO
	- 门槛：至少1个任务稳定训练并完成消融
- M12 SAC/TD3 + MuJoCo任务封装
	- 门槛：可训练、可回放、可导出

对应大纲：RL/DL理论与实践、论文阅读、Sim2Real预热

### 阶段5（M13-M15）DRL算法深化期（上半）

目标：从基础DRL扩展到高级算法，建立算法深度与工程稳定性。

**新内容：M13-M19 完全重构为DRL算法深度学习路线，不涉及ROS2**

- M13 TRPO与信任域策略优化
	- 重点：KL散度约束、自然梯度、Fisher信息矩阵、单调性保证
	- 门槛：TRPO实现可复现，能解释与PPO的理论与实践差异

- M14 分布式RL与多工作流采样
	- 重点：VecEnv、异步梯度更新、样本延迟处理、IPC优化
	- 门槛：3倍样本吞吐提升，异步≈同步最终性能

- M15 视觉强化学习与表示学习（原M18）
	- 重点：数据增强（DrQ/RAD）、对比学习、表示质量评估
	- 门槛：端到端视觉RL收敛，增强明显提升样本效率（>30%）

对应大纲：RL/DL理论与实践、视觉学习、分布式训练、论文阅读

### 阶段6（M16-M19）DRL算法深化期（下半）+ 融合准备

目标：掌握离线RL、模型学习、多智能体与元学习，为融合与Sim2Real奠定基础。

- M16 离线强化学习与批量数据优化
	- 重点：保守Q学习（CQL/IQL）、分布漂移处理、D4RL基准
	- 门槛：竞争性D4RL性能，能解释离线RL的核心挑战

- M17 世界模型与基于模型的学习（原M15）
	- 重点：VAE、RSSM、潜空间规划、轨迹想象
	- 门槛：世界模型预测10+步不爆炸，基于模型学习降低样本需求50%

- M18 多智能体强化学习基础（原M17）
	- 重点：价值分解（QMIX）、CTDE框架、通信与协作
	- 门槛：3+智能体稳定收敛，能分析涌现行为与竞争/协作动力学

- M19 元学习与少样本RL
	- 重点：MAML（梯度式）、RL²（递归式）、快速适应
	- 门槛：少样本适应超越随机初始化，理解内外层动力学

对应大纲：RL/DL理论与实践、模型学习、多智能体、元学习、论文阅读

### 阶段7（M20-M21）ROS2系统集成与Sim2Real准备期

目标：将DRL与控制融合到机器人系统中，建立Sim2Real框架。

- M20 ROS2系统集成与感知-决策-控制闭环
	- 重点：多节点通信、视觉感知集成、RL与控制器统一接口
	- 门槛：RL/控制器可切换运行，视觉驱动决策演示

- M21 Sim2Real与域随机化
	- 重点：仿真→真机的策略迁移、域随机化技术、鲁棒性验证
	- 门槛：Sim2Real实验框架v1，真机演示基本成功

对应大纲：ROS2、Sim2Real、控制+RL融合、论文阅读

### 阶段8（M22-M24）工程化、优化与成果收敛期

目标：提升系统可复现性、性能与硬件落地能力，形成职业化交付。

目标：形成前沿方向能力与职业化交付。

- M22 VLA/具身前沿复现
	- 门槛：复现1篇核心论文并给出落地结论
- M23 系统冻结
	- 门槛：v2.0全链路稳定演示
- M24 作品集封装
	- 门槛：2个完整项目包（代码+报告+视频+讲解）

对应大纲：VLA、论文阅读、系统化表达

## 5. 每天5小时执行模板

- 90分钟：单主题学习（当天只学1个核心概念）
- 120分钟：唯一代码任务（防止任务分散）
- 60分钟：至少1组对比/消融实验
- 30分钟：复盘（结论、问题、明天第一步）

当日红线：
- 1次可复现实验
- 1条明确结论
- 1次代码提交

## 6. 每周执行模板

- 周一：定义唯一目标，跑基线
- 周二-周三：编码与训练
- 周四：对比/消融
- 周五：复盘、修bug、写周报
- 周六：补短板（可选2-4h）
- 周日：休息或轻阅读

## 7. 防跑偏规则

- 本周无实验结果，下周学习时间减30%，转实现与调参
- 同一问题卡3小时，立刻降级为最小可运行版本
- 每周至少2次盲写核心逻辑
- 每月底只检查一件事：本月门槛是否达成
