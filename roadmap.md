# 两年学习地图（按关键部分重构版）

定位：
- 风格：先补基础，再做闭环，再做融合，最后做面向岗位的成果收敛
- 强度：每天5小时
- 主线优先级：前期打牢控制与工程，中期补齐感知与ROS2，后期聚焦规划、操作、Sim2Real 和 VLA
- 约束：每月必须有可复现实验结果与可展示产物

## 1. 总目标（24个月）

24个月后达到：
- 至少独立实现并稳定训练3类DRL算法（DQN、PPO、SAC/TD3）
- 能独立完成PID/LQR并理解MPC，形成统一评估体系
- 能搭建感知-定位-规划-控制闭环（ROS2+仿真+控制/学习策略可切换）
- 形成2个完整项目包（代码+报告+视频+复现实验），且能直接用于目标公司岗位沟通

## 3. 内在联系与先后顺序

核心依赖链：
- 数学 + Python/C++ -> 控制理论/机器人软件 -> ROS2系统集成 -> 3D感知/定位 -> 规划与操作 -> Sim2Real -> VLA与成果收敛
- 视觉是中间层能力：为ROS2感知节点、定位、规划与 VLA 输入提供统一基础
- 嵌入式在“能跑通系统后”发力，避免前期被硬件细节拖慢系统主线

并行策略：
- 主线A（算法）：RL/DL -> 视觉表示 -> 具身策略学习
- 主线B（系统）：控制 -> ROS2 -> 3D感知 -> 规划/操作 -> 仿真与真机
- 辅线C（研究）：论文阅读全程跟进，驱动阶段选题和实验设计

## 4. 24个月路线（按依赖重排）

### 阶段1（M1-M3）基础重建与控制基础期

目标：建立可编码、可实验、可复盘的最低工程能力 + 控制论直觉。

**时间压缩亮点：M1-M2 合并完成原 M1-M3 的核心，提前1个月进入控制领域**

- M1 数学与Python工程基础速成（6周）
	- W1-2：线性代数 + 概率 + Numpy
	- W3-4：C++ 与数据结构库
	- W5-6：工程工作流与可复现性
	- 门槛：手写线性回归、修复bug、工程模板可用

- M2 深度学习与训练工程速成（6周）
	- W1-2：PyTorch 完整训练循环 + 排错SOP
	- W3-4：分类任务 + 数据增强 + 错例分析
	- W5-6：工程规范与快速迭代框架
	- 门槛：单模型稳定训练、新任务30分钟接入

- M3 控制论基础入门
	- 重点：系统建模、PID控制、性能指标
	- 门槛：同一对象完成PID调参，输出指标对比表

对应大纲：数学、Python/C++、深度学习基础、控制论基础、论文阅读（入门）

### 阶段2（M4-M6）控制深化与数据驱动思维期

目标：掌握从PID→LQR→MPC的控制演进路线，为DRL奠定思想基础。

**合并亮点：M4-M5 合并为单一对象上的PID+LQR对比，M6 新增作为控制→DRL桥梁**

- M4 PID + LQR 控制系统完整设计（合并原M4-M5）
	- W1：建模与PID深入（反饱和、离散化）
	- W2：LQR实现与5组Q/R配置对比
	- 门槛：能清楚说明PID与LQR各自优缺点，独立设计LQR

- M5 MPC与约束优化控制
	- 重点：滚动优化、约束处理、QP求解
	- 门槛：PID vs LQR vs MPC 三方案对比，含约束场景分析

- M6 系统辨识与数据驱动学习
	- 重点：参数辨识（LS）vs 黑盒学习（神经网络）
	- 门槛：形成"参数模型 vs 黑盒学习"的权衡认识

对应大纲：控制理论+运动学动力学、系统辨识、神经网络基础、论文阅读

### 阶段3（M7-M9）深度学习工程加速期

目标：从"会调用模型"升级到"会设计、会训练、会诊断、会快速迭代"。

**加速亮点：M7-M8 压缩至5周（从8周），M9 改为基础架构设计而非工程自动化**

- M7 PyTorch训练体系加速版（3周核心）
	- 重点：优化器对比、学习率调整、混合精度、排错能力
	- 门槛：30分钟快速接入新数据、15分钟定位NaN/显存溢出

- M8 计算机视觉任务加速版（2周迭代）
	- 重点：分类基线→错例分析→针对性改进的快速循环
	- 门槛：2周内形成数据-训练-分析-改进完整闭环

- M9 CNN架构与深度网络设计基础（4周）
	- 重点：VGG→ResNet演进、MobileNet轻量化、SE-Block注意力
	- 门槛：理解架构演进原则，能评估模型的精度-FLOPs权衡

对应大纲：Computer Vision、深度网络设计、RL/DL基础、Python工程

### 阶段4（M10-M12）DRL核心能力期

目标：从经典控制扩展到学习控制，建立调参与消融习惯。

- M10 DQN
	- 门槛：CartPole稳定过阈值，日志完整
- M11 PPO
	- 门槛：至少1个任务稳定训练并完成消融
- M12 SAC/TD3 + MuJoCo任务封装
	- 门槛：可训练、可回放、可导出

对应大纲：RL/DL理论与实践、论文阅读、Sim2Real预热

### 阶段5（M13-M15）机器人感知与系统工程期

目标：把单点算法能力转成机器人公司常见的系统工程能力，先把 ROS2、感知和数据链路打通。

- M13 ROS2基础工程与多节点通信
	- 重点：节点、Topic、Service、Action、日志与启动脚本
	- 门槛：感知-决策-执行三节点稳定运行，能定位通信问题

- M14 2D视觉感知与数据链路
	- 重点：检测、分割、跟踪、数据标注与评估
	- 门槛：形成可复用的视觉感知输入，能服务后续定位/规划

- M15 3D感知与场景表示
	- 重点：点云、坐标变换、深度相机、occupancy / TSDF / 简易建图
	- 门槛：能把 RGB-D / 点云转成可供规划使用的场景表示

对应大纲：ROS2、Computer Vision、数学基础、论文阅读

### 阶段6（M16-M19）规划、操作与任务执行期

目标：把感知结果真正变成机器人可执行动作，形成星海图这类公司更看重的“任务闭环”能力。

- M16 运动规划与轨迹控制
	- 重点：MoveIt2、RRT/PRM、轨迹优化、控制接口
	- 门槛：完成点到点规划与避障演示，能解释规划失败原因

- M17 抓取与操作基础
	- 重点：抓取位姿、末端执行器控制、Pick-and-Place 任务拆解
	- 门槛：完成可重复的抓取或放置闭环

- M18 任务状态机与行为树
	- 重点：状态机、行为树、异常恢复、重试与回退
	- 门槛：任务级流程能稳定重复执行，失败后能自动降级

- M19 具身数据与模仿学习
	- 重点：示教数据、离线策略、行为克隆、数据清洗与回放
	- 门槛：能用小规模示教数据跑通一个可执行策略原型

对应大纲：控制理论+运动学动力学+规划、ROS2、Sim2Real、论文阅读

### 阶段7（M20-M21）Sim2Real与系统优化期

目标：把仿真中跑通的能力迁移到真机，补齐稳定性、鲁棒性和部署能力。

- M20 Sim2Real与域随机化
	- 重点：仿真到真机迁移、域随机化、鲁棒性验证
	- 门槛：形成可复用的迁移实验框架

- M21 系统优化与部署
	- 重点：性能监控、资源约束、日志、启动脚本、恢复策略
	- 门槛：系统可稳定连续演示，出现问题能快速定位

对应大纲：Sim2Real、嵌入式系统、ROS2、论文阅读

### 阶段8（M22-M24）前沿复现与作品集收敛期

目标：补齐 VLA 与前沿方向理解，把前面形成的系统能力收束成面试可展示的项目包。

- M22 VLA方向复现
	- 门槛：复现1篇核心论文并给出落地结论
- M23 场景化任务集成
	- 门槛：把感知、规划、控制、异常恢复集成到一个可演示任务里
- M24 作品集封装
	- 门槛：2个完整项目包（代码+报告+视频+讲解）

对应大纲：VLA、论文阅读、系统化表达

## 5. 每天5小时执行模板

- 90分钟：单主题学习（当天只学1个核心概念）
- 120分钟：唯一代码任务（防止任务分散）
- 60分钟：至少1组对比/消融实验
- 30分钟：复盘（结论、问题、明天第一步）

当日红线：
- 1次可复现实验
- 1条明确结论
- 1次代码提交

## 6. 每周执行模板

- 周一：定义唯一目标，跑基线
- 周二-周三：编码与训练
- 周四：对比/消融
- 周五：复盘、修bug、写周报
- 周六：补短板（可选2-4h）
- 周日：休息或轻阅读

## 7. 防跑偏规则

- 本周无实验结果，下周学习时间减30%，转实现与调参
- 同一问题卡3小时，立刻降级为最小可运行版本
- 每周至少2次盲写核心逻辑
- 每月底只检查一件事：本月门槛是否达成
