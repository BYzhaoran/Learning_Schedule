# Learning_Schedule

这是一个面向具身智能 / DRL / 控制 / ROS2 / 视觉 / VLA 的两年学习工程仓库。仓库的设计目标不是堆资料，而是把学习路线拆成可执行的阶段、月份和日常任务，保证每一步都能落到代码、实验和总结上。

## 目录结构

- [学习计划大纲.md](学习计划大纲.md)：总体模块划分与占比
- [计划.md](计划.md)：两年级别的主路线与阶段目标
- [资源索引.md](资源索引.md)：按模块整理的资源入口与学习顺序
- [月计划/README.md](月计划/README.md)：24 个月月计划索引
- [月计划/M01.md](月计划/M01.md) 到 [月计划/M24.md](月计划/M24.md)：每月具体执行卡片

## 推荐阅读顺序

1. 先看 [学习计划大纲.md](学习计划大纲.md)，确认模块和权重。
2. 再看 [计划.md](计划.md)，理解两年路线和阶段顺序。
3. 然后看 [资源索引.md](资源索引.md)，按阶段选资源。
4. 最后进入 [月计划/README.md](月计划/README.md)，按月执行。

## 执行原则

- 先建立基础，再做系统，再做融合。
- 每个月只抓一个主问题，避免任务发散。
- 每个阶段都要有可复现实验和可展示产出。
- 月计划文件只保留当月差异，不重复通用模板。

## 当前结构特点

- 大纲负责“学什么”。
- 计划负责“先后顺序”。
- 资源索引负责“看什么”。
- 月计划负责“这个月具体怎么做”。

## 适合的使用方式

- 如果你想查看长期路线，先看 [计划.md](计划.md)。
- 如果你想立刻开始本月任务，直接打开 [月计划/README.md](月计划/README.md)。
- 如果你想补某个方向的基础，去 [资源索引.md](资源索引.md) 找对应模块。

## 阶段性开源项目验收（可视化创意版）

下面的项目强调“第一眼能看懂、能被吸引、能复现”。每个阶段都建议做一个可交互或可视化演示页。

| 阶段 | 周期 | 可开源项目（建议仓库名） | 可视化创意亮点 | 验收标准 |
| --- | --- | --- | --- | --- |
| A | M1-M3 | `learning-rate-lab`（Numpy实现回归+训练过程可视化） | 滑条调学习率，实时展示损失曲线与参数收敛动画 | 一键启动可视化页面，至少3组超参数对比可复现 |
| B | M4-M6 | `control-art-studio`（PID/LQR/MPC对比） | 同一目标轨迹下三控制器“光绘轨迹图”与误差热力图 | 统一指标表+轨迹动画+对比报告 |
| C | M7-M9 | `vision-error-gallery`（YOLO错例闭环） | 错检/漏检画廊、PR曲线面板、数据增强前后对比滑块 | 数据版本清晰，2轮以上再训练有可视化证据 |
| D | M10-M12 | `rl-arcade-arena`（DQN/PPO/SAC/TD3） | 智能体回放长廊、奖励曲线时间轴、策略失败瞬间集锦 | 至少2类算法稳定复现+消融实验+回放视频 |
| E | M13-M15 | `ros2-digital-twin-show`（感知-决策-控制） | RViz + Web仪表盘双视角，实时显示消息流与控制状态 | 多节点稳定、录包回放完整、闭环演示可重复 |
| F | M16-M18 | `hybrid-policy-observatory`（融合策略） | 纯RL/纯控制/融合策略三屏对照，失败案例地图 | 20组以上标准化对比+鲁棒性统计+失败复盘 |
| G | M19-M21 | `robot-ml-mission-control`（工程化工具链） | 训练任务看板、CI状态墙、实验追踪仪表盘 | CLI工具链、测试链路、自动报告可稳定运行 |

### 展示形式建议（强烈推荐）

- 主页首屏：15-30秒短视频/GIF，先展示“效果”再讲“方法”。
- 对比面板：至少包含“基线 vs 改进版”双栏可视化。
- 失败案例区：专门展示失败场景与修复路径，提升可信度。
- 交互入口：提供最小可试玩 Demo（如 Streamlit/Gradio/Web 可视化）。
- 结果资产：统一导出 `figures/` 与 `videos/`，保证可复用与可传播。

### 开源交付最低标准（每个阶段通用）

- 必备文件：`README.md`、`LICENSE`、`requirements.txt`（或环境说明）、`CHANGELOG.md`
- 必备目录：`src/`、`configs/`、`scripts/`、`experiments/`、`docs/`
- 必备脚本：`train`、`eval`、`reproduce`（至少提供一种一键复现实验命令）
- 必备内容：实验配置、随机种子、主要结果表格、失败案例与已知问题
- 可选加分：Dockerfile、GitHub Actions、英文版README、Demo视频

### 推荐发布节奏

1. 阶段中期：发布 `v0.x`（先保证最小可运行）。
2. 阶段结束：发布 `v1.0`（补齐文档、实验、复现说明）。
3. 跨阶段迭代：新增 `benchmark` 和 `ablation`，保持项目持续更新。

## 备注

仓库内容会随着执行过程持续调整，优先保持路线清晰、任务可落地、输出可复现。
