# M09 DL工程化自动化

## 本月定位
- 主题：围绕 DL 实验工程的核心技术栈做强化实现。
- 前置依赖：M07-M08 的 PyTorch 训练代码可正常训练与评估。
- 本月目标：掌握配置管理、实验追踪、分布式训练、结果分析四类关键技术。

## 本月必学技术栈

### 1. 配置与参数管理
1. Hydra + OmegaConf：层级配置、默认组合、命令行覆盖、多运行目录。
2. Pydantic：训练配置结构化校验，避免字段错拼和类型漂移。
3. JsonSchema：导出配置约束，给脚本和协作成员统一参数规范。
4. Jinja2：生成批量实验配置模板（例如学习率、batch、数据增强组合）。

### 2. 训练性能与稳定性
1. PyTorch AMP：自动混合精度与 GradScaler，降低显存占用。
2. Gradient Checkpointing：以算换存，支持更大模型或更大输入。
3. torch.compile：优化前向与反向图执行性能（PyTorch 2.x）。
4. FSDP / DDP：多卡训练并理解参数分片与通信开销。

### 3. 数据与实验版本化
1. DVC：数据版本、模型版本、管线缓存与远程存储。
2. Git LFS：大权重文件管理，与 Git 提交绑定版本快照。
3. MLflow Tracking：参数、指标、模型工件统一记录。
4. W&B Artifacts：数据集和模型 lineage 追踪，支持回溯与对比。

### 4. 指标计算与可视化
1. TorchMetrics：分类/检测/分割常见指标的统一实现。
2. scikit-learn metrics：AUC/F1/混淆矩阵等离线评估补充。
3. pandas + polars：批量实验结果聚合、排序和统计检验。
4. matplotlib + seaborn + plotly：静态图与交互图双路线报告。

### 5. 训练入口与命令系统
1. Typer：类型安全 CLI，自动帮助文档。
2. Rich：彩色日志、进度条、异常面板提升可读性。
3. Loguru 或 structlog：结构化日志输出，便于检索与聚合。
4. pytest + pytest-benchmark：关键模块正确性与性能回归基线。

## 主线任务（必做，按技术点）
1. 使用 Hydra 重构配置系统，支持命令行覆盖和多组配置组合。
2. 在训练代码接入 AMP + Gradient Checkpointing，并做显存/速度对比。
3. 用 MLflow 或 W&B 记录参数、指标、模型工件。
4. 用 TorchMetrics + pandas 构建统一评估与排行榜脚本。
5. 用 Typer + Rich 重构命令行入口与日志系统。

## 关键技术实验
1. AMP 开关实验：比较吞吐、显存峰值、最终指标。
2. DDP 与单卡实验：比较线性加速比与通信开销。
3. Hydra Sweep 实验：同一任务自动搜索 20 组超参并输出 Top-K。
4. MLflow 与本地日志对比：比较追踪完整度与排障效率。

## 交付物
1. 基于 Hydra 的配置系统（含 schema 校验与模板样例）。
2. 支持 AMP/DDP 的训练脚本。
3. 标准化指标评估模块与排行榜脚本。
4. 可视化报告模板（静态 + 交互各一版）。
5. 可复现实验记录（配置、commit、模型、指标）。

## 完成标志
1. 任意实验都可通过 Hydra 命令行参数直接重跑。
2. 训练脚本至少支持单卡与多卡两种模式。
3. 排行榜脚本可自动生成 Top-K 实验与关键图表。
4. 任一结果都能回溯到数据版本、配置版本、代码版本。

## 常见坑提醒
1. AMP 下 loss NaN 常见于缩放策略不当，需监控 GradScaler 状态。
2. DDP 评估阶段若不做 gather，会造成指标统计偏差。
3. Hydra 覆盖参数命名不一致会导致“看似生效实则未生效”。
4. 只记录最终指标不记录中间曲线，会丢失训练退化信号。
5. 只做本地日志不做工件版本化，后续无法严谨复现。
