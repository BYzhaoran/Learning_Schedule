# M12 SAC/TD3 + MuJoCo任务封装

## 本月定位
- 主题：离策略算法、连续控制、环境封装
- 前置依赖：把 DRL 从离散任务推进到连续控制。
- 本月目标：掌握高维连续动作下的稳定训练与工程封装能力。

## 必学知识点

### 1. SAC 核心机制
1. 最大熵强化学习目标。
2. 双 Q 网络抑制过估计。
3. 温度参数 $\alpha$ 自动调节逻辑。

### 2. TD3 核心机制
1. 双评论家与延迟策略更新。
2. target policy smoothing 的稳定作用。
3. 连续动作噪声注入策略。

### 3. MuJoCo 工程实践
1. 观测归一化与奖励缩放。
2. 回放缓存结构设计（n-step 可选）。
3. 训练、评估、回放、导出的一体化脚本组织。

## 主线任务（必做）
1. 实现 SAC 或 TD3 并完成基础训练。
2. 在同一 MuJoCo 任务上完成 SAC 与 TD3 对比。
3. 封装训练、评估、回放、导出工具链。
4. 记录关键超参对稳定性和样本效率影响。

## 扩展任务（建议做）
1. 增加 HER 或奖励重塑实验（适用于目标型任务）。
2. 做 domain randomization 小实验观察泛化。
3. 尝试把策略迁移到新但相近任务做 zero/few-shot 评估。

## 关键实验
1. SAC 与 TD3 在同任务上做对比（收敛速度、最终回报、稳定性）。
2. 不同 replay buffer 大小与 batch size 组合实验。
3. 不同随机种子下至少 3 次重复实验并统计方差。

## 交付物
1. MuJoCo 训练与回放工具链。
2. SAC/TD3 对比报告与可视化图表。
3. 连续控制任务调参手册。

## 完成标志
1. 可训练、可回放、可导出。
2. 至少一个算法在目标任务上稳定收敛。
3. 能解释 SAC 与 TD3 在你任务中的优劣。

## 常见坑提醒
1. 奖励尺度变化会显著影响 SAC 温度与收敛速度。
2. TD3 中策略更新频率不当容易导致训练抖动。
3. 评估时必须关闭探索噪声，否则结果偏差大。

## 开发板专项（阶段 2：板端推理与控制闭环）
- 目标：把训练好的轻量模型部署到 Jetson Nano 或 Raspberry Pi，完成“感知输入 -> 推理输出 -> 控制命令”闭环。
- 与前后衔接：承接 M06 的板端环境与模板；本阶段的 ROS2 节点与部署脚本在 M18 继续扩展为多节点通信。

### 必学知识点
1. ONNX / TensorRT / TFLite 的基础部署路径（按板型选择）。
2. 板端推理性能分析（延迟、内存、CPU/GPU 占用）。
3. 推理结果到控制命令的接口封装。

### 核心任务
1. 选择 1 个已训练模型并完成板端推理部署。
2. 建立输入数据管线（摄像头或仿真数据）并输出控制指令。
3. 对比 PC 与板端推理延迟和稳定性，形成优化记录。
4. 输出板端部署 README（安装、运行、排错）。

### 完成标志
1. 板端可稳定运行 20 分钟以上不崩溃。
2. 单次推理延迟达到可用水平（按任务自定阈值）。
3. 部署流程可由 README 一键复现。
