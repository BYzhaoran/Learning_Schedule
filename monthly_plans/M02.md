# M02 C++与算法工程习惯

## 本月定位
- 主题：C++基础、STL、调试与工程构建
- 前置依赖：承接 M01 的编程基础，补齐 C++ 实现能力。
- 本月策略：不只“会写”，重点建立工程可维护性与调试闭环。

## 主线任务（必做）
1. 复习 STL 容器、算法与内存管理（vector/map/unordered_map/smart pointer）
2. 完成小型数据结构库（栈、队列、堆、哈希表至少 3 类）
3. 用 CMake 组织项目并加入单元测试
4. 建立统一代码规范（命名、目录、注释、错误处理）

## 必学知识点

### 1. C++ 语言与内存模型
1. 值语义、引用语义、移动语义。
2. RAII、智能指针、资源所有权边界。
3. 常见未定义行为与规避方式。

### 2. STL 与算法复杂度
1. 容器底层结构和适用场景对比。
2. 迭代器失效规则。
3. 哈希冲突、装载因子、重哈希开销。

### 3. 工程构建与调试
1. CMake target 导向写法。
2. Debug/Release 差异与编译选项。
3. gdb/lldb 断点、回溯、watchpoint 基本用法。

## 扩展任务（建议做）
1. 实现 2 种版本（如朴素版与优化版）并做性能对比
2. 接入 clang-format 或等效工具，形成自动格式化流程
3. 用 gdb/lldb 复盘 5 个典型 bug（越界、悬空指针、未初始化）
4. 编写 README，确保新人能 10 分钟编译运行
5. 使用 AddressSanitizer 定位至少 2 个隐性内存问题

## 可选挑战（状态好时做）
1. 引入 benchmark（如 Google Benchmark）做微基准测试
2. 尝试 AddressSanitizer/UBSan 捕捉隐性错误
3. 把一个小模块封装成静态库并复用到第二个 demo

## 关键实验
- 对两种实现方案做性能与复杂度对比。
- 同一算法在不同输入规模下做耗时曲线（例如 $10^3,10^4,10^5$）。
- 对开启/关闭优化选项（-O0/-O2）做对比。
- 增加 1 组 cache 友好布局与普通布局对比。

## 交付物
- C++ 数据结构小库（含基础文档）
- 可编译 CMake 工程（含 debug/release 配置）
- 单元测试（核心模块覆盖）
- 性能对比报告（图表 + 结论）

## 完成标志
1. 修复 10 个以上真实 bug 并形成问题清单。
2. 对至少 3 类常见 C++ bug 能给出定位步骤与修复策略。
3. 能独立搭建可复用的小型 C++ 工程脚手架。

## 每周节奏（建议）
- 第 1 周：STL 与内存基础复盘
- 第 2 周：数据结构实现与测试
- 第 3 周：CMake 工程化与调试
- 第 4 周：性能对比、文档整理、复盘

## 风险与降级方案
- 若语法细节卡住：先过主流程，复杂模板特性放到扩展任务。
- 若工程太重：先单模块跑通，再逐步拆分目录与测试。
- 若测试难推进：先写关键路径 smoke test，再逐步补全。
