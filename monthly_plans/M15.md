# M15 3D感知与场景表示

## 本月定位
- 主题：点云、深度图、几何表示与简单建图
- 前置依赖：完成 M14 的 2D 感知链路，具备基础 Python 和视觉处理能力。
- 本月目标：把图像和深度信息转成规划能用的空间表示。

## 技术主线
1. 从 2D 图像理解走向 3D 场景理解。
2. 认识点云、深度图和体素表示的差异。
3. 建立感知到规划的几何接口。

## 必学技术点

### 1. 3D 感知基础
1. 坐标系、内参、外参和位姿变换。
2. 点云预处理：下采样、滤波、配准。
3. 深度噪声和遮挡问题。

### 2. 场景表示
1. occupancy、TSDF、voxel 的基本直觉。
2. RGB-D 数据到局部地图的转换。
3. 可供规划使用的简易占据地图。

### 3. 与机器人系统联动
1. 3D 感知输出给规划模块。
2. 传感器采样频率与地图更新频率。
3. 低成本传感器下的稳定性权衡。

## 核心任务
1. 用 Open3D 完成点云预处理与可视化。
2. 从 RGB-D 数据生成局部场景表示。
3. 连接 ROS2 节点，输出可供规划使用的地图消息。
4. 写一份 2D 方案与 3D 方案的差异说明。

## 关键实验
1. 体素大小与地图精度的对比。
2. 深度噪声对场景表示的影响。
3. 不同坐标对齐方式下的效果差异。

## 交付物
1. 3D 感知与场景表示模块。
2. RGB-D 到局部地图的转换脚本。
3. 可视化与分析报告。

## 验收标准
1. 能稳定生成场景表示。
2. 能解释点云到地图的关键步骤。
3. 能把 3D 感知结果提供给规划模块。
