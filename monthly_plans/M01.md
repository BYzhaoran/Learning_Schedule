# M01 数学与Python工程基础速成

## 本月定位
- 主题：线性代数、概率基础、Python、C++ 与工程流程的压缩集成
- 前置依赖：无（入门级）。
- 本月策略：6 周集中速成，每周 4 个工作日完成核心闭环。

## 技术主线
1. 线代与概率的数学直觉建立 + Numpy 代码落地。
2. C++ 与工程化思维：从脚本到工程化代码。
3. 工程工作流：可复现实验的最小规范。

## 第 1-2 周：数学与 Python 基础

### 必学知识点（数学）
1. 向量、矩阵、特征值与梯度的几何意义。
2. 条件概率、贝叶斯公式、分布与期望。
3. Numpy 广播机制与向量化。

### 核心任务
1. 整理线代与概率的手推笔记（3 页以内）。
2. 用 Numpy 实现线性回归 + 梯度下降（含可视化）。
3. 记录学习率与 batch size 对收敛的影响。

### 关键实验
- 学习率对比（0.01、0.001、0.0001）的损失曲线。
- Batch size 对比（32、64、128）的收敛速度。
- 重复 3 次实验，记录波动区间。

### 交付物
- `linear_regression_numpy.py` + 实验记录表。
- 3 张 loss 曲线图。

## 第 3-4 周：C++ 与工程构建

### 必学知识点
1. 值语义、引用语义、智能指针的实践。
2. STL 容器与算法的复杂度特性。
3. CMake 与项目组织。

### 核心任务
1. 实现 3-4 个数据结构（vector、map、heap）。
2. 用 CMake 组织项目，接入单元测试。
3. 用 gdb 定位 2 个典型 bug。

### 关键实验
- 朴素实现 vs 优化版本的性能对比（$10^3, 10^4, 10^5$ 数据规模）。
- -O0 vs -O2 编译选项的性能差异。

### 交付物
- C++ 数据结构小库（含 CMakeLists.txt）。
- 性能对比报告（表格 + 图）。

## 第 5-6 周：工程工作流与快速迭代

### 必学知识点
1. 项目目录结构规范（src/、configs/、scripts/、logs/）。
2. 配置、代码、数据版本的绑定。
3. 实验记录模板与可复现性。

### 核心任务
1. 统一项目结构并建立命令入口。
2. 实现简单日志与配置管理。
3. 用工程模板重跑 M01 前半的线性回归，验证流程有效。

### 关键实验
- 同一配置重复 3 次，检查结果一致性。
- 模拟失败场景（参数错误），验证排错能力。

### 交付物
- 工程模板 v1（包含训练脚本、配置、日志）。
- 改造前后对比说明。

## 完成标志
1. 能手推线代/概率概念，代码可复现。
2. 能快速搭建 C++ 小工程并定位 bug。
3. 新任务 30 分钟内可接入并跑出结果。
