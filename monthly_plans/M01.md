# M01 数学速成 + Python基础

## 本月定位
- 主题：线性代数、概率基础、Python与Numpy实现
- 前置依赖：对应 [learning_plan_outline.md](../learning_plan_outline.md) 中的数学基础速成补全与 Python 持续学习。
- 本月策略：以“数学理解 + 代码实现 + 实验记录”三线并行，避免只刷课不落地。

## 主线任务（必做）
1. 学完线代核心内容并整理手推笔记（向量、矩阵、特征值、梯度）
2. 用 Numpy 实现线性回归与梯度下降（含可视化）
3. 记录学习率与 batch size 对收敛的影响
4. 形成一份“数学概念 -> 代码实现”对照表

## 必学知识点

### 1. 线性代数核心
1. 向量空间、线性相关、基与维度。
2. 矩阵分解直觉（特征分解、SVD 的用途）。
3. 梯度、Jacobian、Hessian 在优化中的角色。

### 2. 概率统计基础
1. 条件概率、贝叶斯公式、独立性与相关性。
2. 期望、方差、协方差与偏差-方差直觉。
3. 常见分布（高斯、伯努利、二项）与建模含义。

### 3. Python 数值计算实践
1. Numpy 广播机制与向量化思维。
2. 时间复杂度与空间复杂度的基本权衡。
3. 可复现实验的最小规范（随机种子、配置、日志）。

## 扩展任务（建议做）
1. 手写 logistic regression（二分类）并比较与线性回归差异
2. 用纯 Python 和 Numpy 分别实现同一算法，对比速度与可读性
3. 用 matplotlib 画出 loss 曲线、参数轨迹、学习率敏感性图
4. 选 2 篇入门博客或教程做复现实验
5. 增加 L1/L2 正则化对过拟合影响对比

## 可选挑战（状态好时做）
1. 不依赖 sklearn，从零实现 train/val split、标准化、指标评估
2. 封装一个最小实验框架（config + logger + seed）
3. 用一个真实小数据集（如房价、酒质）完成端到端实验

## 关键实验
- 同一数据集做 3 组学习率对比，输出损失曲线。
- 同一学习率下做 3 组 batch size 对比，输出收敛速度与最终误差。
- 固定配置重复 3 次，记录波动区间，建立“随机性意识”。
- 增加 1 组特征缩放前后收敛速度对比。
- 增加 1 组初始化方式（零初始化/随机初始化）对比。

## 交付物
- `linear_regression_numpy.py`（或同等实现文件）
- 实验记录表（学习率、batch size、初始值、结果）
- 可视化图（loss 曲线至少 3 张）
- 月总结（本月最重要的 3 个结论 + 下月改进）

## 完成标志
1. 能独立解释梯度下降收敛过程，代码可复现。
2. 给一个新数据集，能在 2 小时内跑出 baseline 并解释结果。
3. 能从数学定义推导到代码实现并指出近似误差来源。

## 每周节奏（建议）
- 第 1 周：线代与概率核心补齐 + 笔记整理
- 第 2 周：线性回归手写与调试
- 第 3 周：对比实验与可视化
- 第 4 周：复盘、重构、补短板

## 风险与降级方案
- 若数学推进慢：先保证“能解释 + 能写代码”最小闭环，推导细节后补。
- 若代码卡住超过 3 小时：退回最小可运行版本，先跑通再优化。
- 若实验结果混乱：固定随机种子、减少变量、一次只改一个超参。
